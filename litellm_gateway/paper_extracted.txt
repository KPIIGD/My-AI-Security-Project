


한국어 LLM Gateway에서 변이형 PII 우회 저감을 위한 LLM-free Layer 0 Guardrail 설계와 평가
양유상1*, 김민우2, 정연서2 (발표자에 * 표시)
중부대학교1, CCIT연구소2 

English Title of Paper Presented in 2026 Conference of The Conference Security(스타일 논문 영문제목)
YuSsang Yang1*, Gil-Dong Koh2, Gildong Kim2 (발표자에 * 표시)

요약 : 한국어 LLM Gateway에서는 자모·Unicode·표기 변형으로 텍스트형 개인정보 탐지가 우회될 수 있다. 본 논문은 한국어 정규화와 PII 사전/패턴 탐지를 결합한 LLM-free Layer 0를 pre-call 계층으로 배치한다. 10k stratified payload 평가에서 TRUE detection은 Layer 0 포함 구성 94.32%, LLM judge 포함 구성 90.96%였고, KR_semantic에서는 96.39% 대 87.40%였다. 또한 p99 지연은 830ms 대 4,819ms로 낮아, Layer 0의 early blocking이 한국어 PII 사전 차단에 효과적임을 보였다.
  
Key Words : Korean PII, LLM Gateway, Layer 0 Guardrail, Korean Normalization, Pre-Call Blocking

1. 서 론

<표>

대규모 언어모델(LLM)을 활용한 서비스에서는 사용자의 입력과 모델의 출력에서 개인정보가 포함될 가능성이 높다. 특히 LLM Gateway는 다양한 모델 호출이 집중되는 지점이므로, 개인정보 입력을 사전에 차단하거나 마스킹하는 보안 계층이 필요하다. 그러나 한국어 개인정보는 주민등록번호나 전화번호처럼 정형화된 패턴뿐 아니라 진단명, 알레르기, 종교, 직위, 소속, 결혼상태 등 텍스트형·의미형 개인정보를 포함한다. 이러한 정보는 자모 분해, Unicode 변형, 초성 표기, 야민정음, 로마자 혼합, 구분자 삽입 등으로 표면형을 쉽게 바꿀 수 있어 기존 탐지기를 우회할 수 있다.
기존 PII 탐지 연구와 guardrail 시스템은 정규식, NER, 관리형 보안 필터, LLM judge 등을 활용해 개인정보를 탐지한다. 그러나 범용 탐지기는 영어 중심 엔티티와 정형 패턴에 강한 반면, 한국어 텍스트형 PII와 표기 변형에 충분히 최적화되어 있지 않을 수 있다. 또한 LLM judge는 의미 판별 능력이 있으나 외부 API 호출, 비용, 지연, 인터넷 의존성, 재현성 문제가 발생한다. 따라서 한국어 LLM 서비스에서는 모델 호출 전에 한국어 입력을 정규화하고, 개인정보 사전 및 패턴을 이용해 선행 차단하는 경량 계층이 필요하다.
본 논문은 한국어 LLM Gateway에서 동작하는 LLM-free Layer 0 PII Guardrail을 설계하고 평가한다. 제안 방식은 사용자 입력을 LLM 및 외부 guardrail로 전달하기 전에 한국어 정규화기를 적용하고, 이후 PII 사전/패턴 탐지를 수행하여 PASS, BLOCK, MASK 정책을 결정한다. 특히 본 논문은 Layer 0가 기존 LLM judge 기반 구성보다 한국어 텍스트형 PII에서 더 높은 탐지율과 낮은 지연을 보이는지 실험적으로 분석한다.
본 논문의 주요 기여는 다음과 같다. 첫째, 한국어 자모·Unicode·표기 변형을 고려한 LLM-free Layer 0 PII Guardrail을 설계하였다. 둘째, 10k stratified payload와 KR_semantic slice에서 기존 LLM judge 기반 구성과 head-to-head 비교하였다. 셋째, TRUE detection, real bypass, p99 latency, false positive, McNemar 검정, ablation을 함께 제시하여 운영형 LLM Gateway에서의 실용성을 평가하였다.

2. 본 론
2.1 관련 연구 및 위협모델
한국어 개인정보 탐지는 한글 자모, 호환 자모, Unicode 정규화, 표기 변형의 영향을 받는다. 기존 한글 정규화 연구는 유니코드 환경에서 한글 표현 체계와 조합 문제를 분석하였으며[1], 한국어 PII 연구는 개인식별번호와 개인정보 처리 능력 평가를 다루었다[2]. 그러나 기존 연구는 주로 정규화 자체, 데이터셋 구축, 모델 기반 탐지 성능에 초점을 두었고, LLM Gateway의 pre-call 단계에서 deterministic guardrail을 배치하여 지연·비용·오탐까지 함께 평가한 연구는 제한적이다.
문자 수준 공격 연구는 invisible character, homoglyph, reordering 등 인코딩 기반 교란이 NLP 시스템에 실질적인 우회 수단이 될 수 있음을 보였다[3]. 또한 최근 guardrail robustness 연구는 typo, camouflage, cipher, veiled expression 등 다양한 mutation이 guardrail 모델의 강건성을 저하시킬 수 있음을 보고하였다[4]. 한국어는 자모와 음운 단위 변형이 가능하므로, 영어 중심 입력 검증과 다른 위협모델이 필요하다[5]. 본 연구는 이러한 위협을 한국어 텍스트형 PII 우회 문제로 정의하고, LLM Gateway 앞단에 Layer 0를 배치해 선행 차단 효과를 평가한다.
2.2 제안 방법: Layer 0 Pre-call Guardrail
제안 Layer 0는 사용자 입력이 LLM과 외부 guardrail로 전달되기 전에 실행되는 pre-call 계층이다. 먼저 한국어 normalizer가 Unicode, 자모, 초성, 야민정음, 숫자·구분자 변형을 표준형으로 복원한다. 이후 한국어 PII detector가 regex와 키워드/의미형 사전을 이용해 개인정보를 탐지한다. 탐지 결과에 따라 요청은 PASS, BLOCK, MASK 중 하나로 처리된다.
Layer 0의 핵심은 LLM 호출 없이 로컬에서 동작한다는 점이다. LLM judge는 복잡한 의미 판별에 유용하지만, 모든 입력에 대해 호출하면 지연과 비용이 증가한다. 반면 Layer 0는 한국어 변이형 입력을 먼저 canonical form으로 정규화하고, 한국어 PII 사전/패턴을 적용하므로 한국어 텍스트형 PII를 빠르게 차단할 수 있다. 이 구조는 LLM judge를 완전히 대체하기보다, judge 이전에 배치되는 저비용·저지연 선행 방어선으로 설계되었다.
Fig. 1. Architecture of Layer 0 Pre-Call Guardrail in Korean LLM Gateway
사용자 입력 → Layer 0 Korean Normalizer → Korean PII Detector → L1 Presidio → L2 Bedrock Guardrails → L3 Lakera → LLM → L4 GPT-4o-mini Judge
Fig. 1은 제안 구조의 처리 흐름을 나타낸다. Layer 0는 입력 단계에서 한국어 변이형 PII를 정규화하고 탐지하며, 이후 기존 다계층 guardrail stack으로 입력을 전달한다. Full stack 구성에서는 Layer 0와 L4 judge가 함께 사용되어 상호보완적인 방어 효과를 제공한다.
2.3 실험 설계
본 연구는 10k stratified payload를 사용하여 네 가지 구성을 비교하였다. 주요 slice로는 한국어 텍스트형 PII를 포함하는 KR_semantic slice(n=1,302)를 별도로 분석하였다. 평가 지표는 TRUE detection, real bypass, KR_semantic TRUE detection, KR_semantic bypass, p99 latency를 사용하였다. 또한 B와 C의 차이를 확인하기 위해 McNemar 검정을 사용하였고, 정상 텍스트 false positive와 Layer 0 ablation도 함께 검토하였다.

TRUE detection은 ground-truth PII payload의 차단·마스킹 성공률, real bypass는 해당 payload가 차단·마스킹 없이 통과한 비율을 의미한다. KR_semantic은 한국어 텍스트형·의미형 PII slice이며, p99 latency는 전체 요청 지연의 99번째 백분위 값이다. Smart Skip은 L0~L3에서 차단·마스킹이 확정된 요청에 대해 L4 LLM judge 호출을 생략하는 운영 최적화 정책이다.

실험 결과, Layer 0를 포함한 C 구성은 LLM judge를 포함한 B 구성보다 전체와 KR_semantic 모두에서 높은 TRUE detection을 보였다. 전체 10k 기준으로 C는 94.32%, B는 90.96%였으며, C가 B보다 3.36%p 높았다. 특히 KR_semantic slice에서는 C가 96.39%, B가 87.40%로 나타나 +8.99%p의 차이를 보였다. 이는 한국어 텍스트형 PII에서 Layer 0의 한국어 정규화와 사전/패턴 기반 탐지가 효과적임을 의미한다.
지연 측면에서도 C 구성은 B 구성보다 우수하였다. 지연 결과는 단순히 계층 수가 많을수록 느리다는 방식으로 해석하면 안 된다. 표면적으로는 C가 A에 Layer 0를 추가한 구성인데도 p99 latency가 A의 1,317ms에서 C의 830ms로 감소하였다. 이는 평가 파이프라인이 모든 요청을 모든 계층에 끝까지 전달하는 고정 순차 구조가 아니라, 앞단에서 BLOCK 또는 MASK 결정이 내려지면 downstream guardrail 호출을 줄이는 cascade 방식으로 동작하기 때문이다. Layer 0는 한국어 텍스트형 PII를 로컬 pre-call 단계에서 먼저 탐지하므로, 일부 위험 입력이 L1~L3 후속 경로까지 진행하지 않아 tail latency가 감소한다. 반면 B는 L4 LLM judge 호출 지연이 tail latency에 반영되어 p99가 4,819ms로 높게 나타난다.
B와 C의 paired 비교에서 McNemar p-value는 2.04e-28로 보고되어, Layer 0 포함 구성의 개선이 우연한 차이로 보기 어렵다. 정상 텍스트에 대한 false positive는 2%로 보고되었으며, 이는 제안 방식이 공격성 입력뿐 아니라 정상 입력에 대해서도 비교적 안정적으로 동작함을 시사한다.
다만 Layer 0의 효과를 정규화 단독의 효과로 해석해서는 안 된다. ablation 결과에서 KR_semantic 기준 정규화 단독 gain은 0.31%p로 제한적이며, detector/dictionary의 기여가 더 크게 나타났다. 따라서 본 연구의 주장은 “정규화만으로 성능이 향상되었다”가 아니라, “정규화-우선 구조가 한국어 PII 사전/패턴 탐지를 가능하게 하고, 이를 pre-call 계층으로 배치했을 때 한국어 텍스트형 PII 우회를 줄일 수 있다”는 것이다.
2.5 논의
C가 B보다 높은 결과를 보인 것은 Layer 0가 LLM judge를 모든 경우에 대체한다는 의미가 아니다. 오히려 한국어 텍스트형 PII 중 상당 부분은 LLM 호출 전에 deterministic하게 탐지·차단할 수 있음을 보여준다. LLM judge는 복잡한 문맥 판단에 유용하지만, 모든 요청에 대해 호출할 경우 지연과 비용이 증가한다. 반면 Layer 0는 한국어 변이형 입력을 빠르게 정규화하고, 한국어 PII 사전/패턴을 적용하여 선행 방어선으로 동작할 수 있다.
D가 가장 높은 탐지율을 보인 점은 Layer 0와 L4 judge의 상호보완성을 보여준다. 실운영에서는 Layer 0를 기본 방어선으로 두고, 고위험 또는 불확실 케이스에 L4 judge를 선택적으로 적용하는 구성이 적합하다. 이는 정확도, 비용, 지연을 동시에 고려해야 하는 LLM Gateway 운영 환경에서 실용적인 설계 방향이다.
본 연구의 한계도 존재한다. 첫째, 사용 데이터는 synthetic payload 중심이므로 실제 운영 로그와의 외부 타당성 검증이 필요하다. 둘째, 사전/패턴 기반 탐지는 신규 PII 표현과 도메인 특화 표현에 대해 지속적인 사전 확장이 필요하다. 셋째, 일부 비교 구성은 외부 API와 관리형 guardrail에 의존하므로, 실행 환경과 서비스 설정에 따라 재현성 차이가 발생할 수 있다.

3. 결론
본 논문은 한국어 LLM Gateway에서 변이형 텍스트 PII를 사전 차단하기 위한 LLM-free Layer 0 Guardrail을 설계하고, 10k stratified payload에서 기존 LLM judge 기반 구성과 비교 평가하였다. 실험 결과 Layer 0 포함 구성은 전체 TRUE detection과 KR_semantic slice 모두에서 LLM judge 포함 구성보다 높은 탐지율을 보였고, p99 지연도 낮았다. 특히 KR_semantic에서 +8.99%p의 차이는 한국어 정규화와 사전/패턴 기반 pre-call 탐지가 한국어 텍스트형 PII 방어에 효과적임을 보여준다.
다만 제안 방식은 synthetic payload 기반 평가와 사전 기반 탐지의 확장성이라는 한계를 가진다. 향후 연구에서는 실제 한국어 운영 로그에 가까운 clean corpus, attack-family별 ablation, 한국어 도메인별 PII 사전 확장, Smart Skip 기반 L4 호출 최적화를 추가 검증할 필요가 있다. 본 연구는 한국어 LLM 서비스에서 deterministic pre-call guardrail이 저지연 개인정보 방어선으로 활용될 수 있음을 보이며, LLM Gateway 보안 아키텍처 설계의 실증적 근거를 제공한다.
