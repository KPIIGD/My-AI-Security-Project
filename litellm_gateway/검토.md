## 3. 본문 — 강점 (유지할 부분)

- **문제 정의가 명확합니다.** "한국어 변이형 텍스트 PII 우회"에서 "LLM-free Layer 0"로 이어지는 인과 구조가 분명합니다.
- **A/B/C/D 4구성 비교**와 **McNemar 검정 (p = 2.18e-22)** 으로 통계적 유의성을 확보했습니다.
- **한계를 솔직히 인정**하고 있습니다. synthetic 데이터의 일반화 한계, FP 0건의 해석 주의, 정규화 단독 gain 0.31%p의 제한성을 모두 명시한 점은 심사 시 긍정적으로 평가될 부분입니다.
- **정책적 함의**를 다루고 있습니다. 감사 로그 최소 보존, 약관 기반 제재 등은 한국 학회 톤에 잘 부합합니다.
- **Rule of three**로 95% 상한을 0.03%로 산출한 부분은 통계적으로 정확합니다 (3/10,000 = 0.03%).
- **재현성 논의**가 적절합니다. Layer 0가 deterministic 계층임을 외부 API 의존 구성과 명확히 분리해 설명한 점이 좋습니다.

## 4. 본문 — 수정 권장 사항

### (1) "Smart Skip" 용어 정의 누락

3.4 논의에서 "Smart Skip 정책은 detection impact 없이 L4 호출을 94.32% 줄일 수 있다"라는 문장이 갑자기 등장합니다. 이 용어가 처음 나오는 곳이므로 한 문장으로라도 정의를 추가해야 합니다.

권장 문구 예시:
> "Layer 0-L3에서 BLOCK 또는 MASK가 확정된 케이스에 대해 L4 judge 호출을 건너뛰는 Smart Skip 정책을 적용하면..."

### (2) 약어 표기 방식

NER 부분이 다소 무겁습니다.
- 현재: `NER(Named Entity Recognition; 문장에서 이름·기관·주소 등 개체를 식별하는 기법)`
- 권장: `NER(Named Entity Recognition, 개체명 인식)`

### (3) "10k" vs "10,000건" 혼용

한 문서 안에서 두 표기가 섞여 있습니다. 학회 논문은 통상 **"10,000건"** 으로 통일하거나, 첫 등장 시 `10,000건(10k)`로 표기하고 이후에 약어를 사용하는 방식을 권장합니다.

### (4) A/B/C/D 구성 사전 정의

3.2~3.3에서 `B(L1-L4)`, `C(L0-L3)`, `D(L0-L4)`가 갑자기 등장합니다. Table 1에 정의가 있더라도 본문 첫 등장 시 한 문장으로 보강이 필요합니다.

권장 문구 예시:
> "A는 baseline, B는 L1-L4 외부 가드레일 구성, C는 Layer 0를 추가한 L0-L3 구성, D는 Layer 0와 L4 judge를 모두 포함한 구성이다."

### (5) 어색한 표현

- 현재: "외부 API 비용과 지연을 요구한다"
- 권장: "외부 API 비용과 지연이 추가로 발생한다"

### (6) 부정적 결과의 보강

3.3에서 **transaction(-21.12%p), loan(-13.43%p)** 같은 negative gain은 결정적인 약점인데, "LLM judge의 보완 효과가 나타났고, D 구성에서 회복되었다"라고 한 줄로 처리되어 있습니다. 심사자가 추가 질문할 가능성이 높으므로, **D 구성에서 정확히 몇 %p 회복되었는지** 수치 하나만 추가하면 방어가 훨씬 단단해집니다.

다섯 편 모두 적절하며, 최신성도 양호합니다. 특히 [4] EMNLP 2025와 [5] arXiv 2025를 인용한 점이 좋습니다.

다만 다음 두 가지는 학회 사무국에 한 번 확인이 필요합니다.

- **[3] Boucher et al.**: 현재 "Proceedings of the 43rd IEEE Symposium on Security and Privacy"로 표기되어 있는데, IEEE S&P 2022는 통상 "IEEE S&P 2022" 또는 "SP 2022"로 축약 표기되는 경우가 많습니다.

## 6. 데이터 정합성 확인

프로젝트 초기 메모(2026-04-18 시점)에는 **8,400건 평가, Layer 0 26.0%p 개선(n=1,193)** 으로 기록되어 있었는데, 논문에서는 **10,000건 stratified payload, KR_semantic +8.99%p**로 업그레이드되었습니다.

데이터 규모와 stratification이 더 엄밀해진 점은 긍정적입니다. 다만 평가 스크립트 버전이 그 사이에 바뀌었을 수 있으므로, **최신 평가 스크립트와 원본 로그를 공저자 간에 한 번 더 동기화**해두는 것을 권장합니다. 논문 수치는 평가 스크립트 버전이 고정되어야 재현 가능합니다.
