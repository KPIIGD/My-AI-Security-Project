## ⚠️ 개인정보 / 공개 레포 주의

이 레포는 **공개(Public)** 입니다. 슬랙 대화에는 이름·연락처·계정 등 **개인정보(PII)가 포함될 수 있으므로**,
실제 대화 로그 파일(`YYYY-MM-DD.md`)은 **`.gitignore`로 로컬 전용** 처리되어 깃에 커밋·푸시되지 않습니다.

- ✅ 커밋되는 것: `목차.md`, `히스토리.md`, `_템플릿.md` (개인정보 없는 구조 파일)
- 🚫 커밋 안 되는 것: `YYYY-MM-DD.md` 형식의 실제 대화 로그 (로컬에만 존재)
- LLM은 로컬 파일을 직접 읽으므로, 깃에 없어도 맥락 파악에는 지장 없음.
- 만약 로그도 동기화하고 싶다면 **레포를 비공개로 전환**하거나, 개인정보를 마스킹한 버전만 별도로 관리하세요.

## 🔄 자동 수집 (현재 적용됨 — 30분 주기)

이 머신에 **Windows 작업 스케줄러 작업 `SlackLogFetch`** 가 등록되어, PC가 켜져 있는 동안
**30분마다** [`../scripts/fetch_slack_logs.py`](../scripts/fetch_slack_logs.py) `--days 2` 를 자동 실행합니다.
멱등이라 매번 그날 파일을 갱신하되 📌 요약 블록은 보존합니다 (사실상 실시간 수집).

- **토큰**: 사용자 환경변수 `SLACK_BOT_TOKEN` (스크립트가 Windows 레지스트리에서도 직접 읽음). 코드/깃엔 없음.
- **로그**: `슬랙/_auto_fetch.log` 는 수집 대상 아님(`*.log` → .gitignore). 실제 대화 로그도 로컬 전용.
- **상태 확인**: `schtasks /Query /TN SlackLogFetch /V /FO LIST`
- **주기 변경**: `schtasks /Change /TN SlackLogFetch /RI 15` (분 단위) 또는 재등록
- **일시 중지/재개**: `schtasks /Change /TN SlackLogFetch /DISABLE` / `/ENABLE`
- **삭제**: `schtasks /Delete /TN SlackLogFetch /F`
- **토큰 교체**(재발급 시): `setx SLACK_BOT_TOKEN "xoxp-새토큰"`

> ⚠️ PC가 꺼져 있으면 그 시간대는 수집이 멈춥니다. 24/7이 필요하면 상시 켜진 서버/PC에 등록하세요.

## 작성 가이드

### 방법 A — 수집 스크립트 수동 실행 (MCP 불필요)

```bash
# PowerShell:  $env:SLACK_BOT_TOKEN = "xoxp-..."
export SLACK_BOT_TOKEN=xoxp-...
python scripts/fetch_slack_logs.py            # 오늘, 공개 채널 전체 자동 탐지
python scripts/fetch_slack_logs.py --days 5   # 최근 5일치
```

→ 스크립트: [`../scripts/fetch_slack_logs.py`](../scripts/fetch_slack_logs.py)
→ 요약(📌) 섹션을 채우고, 위 "날짜별 로그 인덱스" 표와 [`히스토리.md`](히스토리.md)를 갱신하세요.

### 방법 B — Slack MCP

```bash
claude mcp add slack -- npx -y @modelcontextprotocol/server-slack
```
(환경변수 `SLACK_BOT_TOKEN`, `SLACK_TEAM_ID` 설정 후 세션 재시작 → "지난주 #general 정리해줘"처럼 요청)

### 방법 C — 수동

[`_템플릿.md`](_템플릿.md)를 복사해 `YYYY-MM-DD.md`로 만든 뒤 직접 채웁니다.

**공통 — 필요한 토큰 scope:** `channels:history`, `channels:read`, `users:read`
(비공개 채널은 `groups:history`, `groups:read` 추가)
**개인정보:** 외부 공유가 필요하면 이름/번호 등은 마스킹 후 별도 파일로.
