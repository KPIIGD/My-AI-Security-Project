1. Base model + tokenizer
━━━━━━━━━━━━━━━━━━━━

- Base: klue/bert-base (BERT 110M, MLM-only)
- Phase 2 ablation 후보: klue/roberta-base
- Tokenizer: BertTokenizerFast (AutoTokenizer)
- 토큰화: wordpiece. 한국어 char-level 우세 → token offset ↔ char offset 1:1
- 라이선스: KLUE CC-BY-SA → 재배포 가능 (Phase 6 alphagyuu/mncai 라이선스 모호함 회피)

2. 출력 방식
━━━━━━━━━━━━━━━━━━━━

BIO token classification (BIOES 아님).
HuggingFace AutoModelForTokenClassification 표준.
후처리에서 B-X I-X* → contiguous span 변환.
span classification 안 함 (KLUE-NER이 BIO 포맷, marginal gain).

3. 모델 label 목록
━━━━━━━━━━━━━━━━━━━━

7개 (BIO):
  O           id=0
  B-NAME      id=1
  I-NAME      id=2
  B-ADDRESS   id=3
  I-ADDRESS   id=4
  B-ORG       id=5
  I-ORG       id=6

의도적 minimal: NAME / ADDRESS / ORG 3 entity만 NER이 학습.
나머지 30+ 종(RRN, PHONE, EMAIL, CREDIT_CARD, RELATION, MEDICAL_RECORD_NO 등)은 regex + dict 담당.

이유:
- 한국어 NER 라벨링 비용 ↑
- 정형 entity는 regex가 NER보다 정확/빠름
- NER이 모든 걸 다루면 FP 폭증 (Phase 6 alphagyuu any-detection 100% 사례)
- v0.2가 detector를 source별 분리 설계


━━━━━━━━━━━━━━━━━━━━
4. label → EntityType 매핑표
━━━━━━━━━━━━━━━━━━━━

NAME    → PERSON_NAME
ADDRESS → ADDRESS_FULL    (ADDRESS_UNIT은 후처리 dict 분기)
ORG     → ORGANIZATION    (SCHOOL/HOSPITAL은 후처리 dict reclassify)

세분 entity (SCHOOL/HOSPITAL/BUSINESS_REG_NO 등):
- NER은 ORGANIZATION으로만 emit
- dict (organization_suffix / school_suffix / hospital_suffix)가 reclassify
- allow_experimental_entities=True 일 때만 NER 세분화 시도 옵션 가능

ADDRESS:
- NER은 ADDRESS_FULL만 (예: "서울시 강남구 테헤란로 123")
- ADDRESS_UNIT (예: "101동 1203호")은 dict + regex 분리
- 복합 ADDRESS는 span resolver가 통합


━━━━━━━━━━━━━━━━━━━━
5. offset 종류
━━━━━━━━━━━━━━━━━━━━

- 모델 raw 출력: subword token offset (BERT wordpiece)
- 학습 입력: char list (한글 1자 = 1 word, is_split_into_words=True)
- 결과: token offset ↔ char offset 사실상 1:1 (한국어 wordpiece가 1 char 안 쪼개짐)


━━━━━━━━━━━━━━━━━━━━
6. raw text → offset 복원 방법
━━━━━━━━━━━━━━━━━━━━

  1. raw_text → char_list = list(raw_text)
  2. enc = tokenizer(char_list, is_split_into_words=True,
                     return_offsets_mapping=True,
                     truncation=True, max_length=256)
  3. word_ids = enc.word_ids()  # 각 subword가 가리키는 char index
  4. forward → logits → argmax → token labels
  5. BIO → contiguous span 결합
  6. (token_start, token_end) → word_ids 통해 (char_start, char_end)
  7. PIISpan(start=char_start, end=char_end,
             text=raw_text[char_start:char_end], ...)

v0.2 계약 보장: span.text == raw_text[span.start:span.end] 100% 성립.
이유: char list가 raw_text의 lossless 표현. word_ids 결정론적 매핑이라 round-trip 손실 없음.

7. 조사 / 어미 / 호칭 모델 포함 여부
━━━━━━━━━━━━━━━━━━━━

제외. 학습 데이터에 NAME/ADDRESS/ORG 본체만 라벨.

예:
  "홍길동이 신청했습니다"           → NER 출력: 홍길동 (3자)
  "김민수에게 연락하세요"           → NER 출력: 김민수 (3자)
  "서울시 강남구 테헤란로 123에..."  → NER 출력: 서울시 강남구 테헤란로 123

근거:
- KLUE-NER PS/LC/OG가 본체만 라벨
- Faker 합성도 entity_text 자체만 라벨링
- 조사(이/가/을/를/에/에게...)는 boundary corrector 분리 책임
- v0.2 josa_rules.yaml 22개 조사 + 4개 honorific + 7개 ending과 호환

예외: 모델이 노이즈로 조사 포함하는 edge case는 boundary corrector가 후처리 분리.

8. confidence / logit 가용성
━━━━━━━━━━━━━━━━━━━━

가능. logits → softmax → token별 7-way 확률.

Span score 옵션:
  A. mean pooling (권장): mean([probs[t, entity_id] for t in span_tokens])
  B. first-token max:     max(probs[span.first_token, entity_ids])

선택: A 권장. B는 짧은 span에서 over-confident.

reason_codes (≥1 필수):
  ner.argmax
  ner.softmax_mean
  ner.span_length_<n>
  ner.calibrated_v1


━━━━━━━━━━━━━━━━━━━━
9. threshold / calibration 계획
━━━━━━━━━━━━━━━━━━━━

1차 (학습 직후): uniform threshold 0.85
2차 (calibration): val set에서 PERSON_NAME 대상 temperature scaling 또는 isotonic regression
3차 (FP tuning): hard_cases_v0 negative + normal_kr 10k에서 FP@threshold 측정 → per-entity threshold 미세조정

v0.2 scoring.yaml 호환:
- score_bands: high_confidence 0.90 / mask_or_review 0.75 / context_judge 0.55
- NER raw score를 [0,1] normalize → 그대로 score band 투입
- low-conf span(<0.55)도 candidate로 emit → dict/context booster가 보강할 기회 제공

calibration metadata 포맷:

  "temperature": 1.23,
  "fitted_on_split": "val_v1",
  "fitted_at": "2026-05-XX",

    "PERSON_NAME": 0.85,
    "ADDRESS_FULL": 0.80,
    "ORGANIZATION": 0.75
  },

10. max sequence length + 긴 문장
━━━━━━━━━━━━━━━━━━━━

- 학습 max_length: 128 char
- inference max_length: 256 char (필요 시 512)
- 긴 문서: sliding window stride 128, overlap score mean pooling, span IoU≥0.5 merge
- 한 문장 > max_length: 어절 경계 split (조사 끝/마침표 기준)
