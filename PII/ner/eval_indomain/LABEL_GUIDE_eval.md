# in-domain 평가셋 라벨 가이드 (eval)

> **기준 문서**: `PII/ner/reports/label_guide_v1.md` (NAME/ADDRESS/ORG 경계 정의 30+ 케이스).
> 본 문서는 **평가셋 라벨링 한정** 추가 규칙 + 마크업 사용법. 충돌 시 label_guide_v1 우선.
> **목적**: 검증 지적 "평가셋 도메인 불일치(KLUE 뉴스 vs 챗 PII)" 대응 — **챗/대화 in-domain** 정답셋을 2인 독립 라벨로 구축.

---

## 1. 라벨 클래스 (3개만)

NER 모델은 **NAME / ADDRESS / ORG** 만 평가한다. (전화/RRN/여권 등 구조형 PII는 regex가 담당 → 별 트랙 `structured_eval.jsonl` 합성으로 자동 평가, 사람 라벨 X.)

| 클래스 | 한 줄 정의 |
|---|---|
| **NAME** | 사람 이름 (실명·예명·핸들). 공인/역사인물·직책단독·호칭단독 제외 |
| **ADDRESS** | 주소 (행정구역+도로명+동/호 full span). 일반 지명·비식별("우리 집") 제외 |
| **ORG** | 조직 (회사·학교·병원·기관). 부서단독·일반명사("회사") 제외 |

→ 상세 경계 케이스는 **label_guide_v1.md** §1~4 필독.

---

## 2. 마크업 방식

annotator 파일의 `annotation` 필드를 **직접 편집**해서 PII를 감싼다:

```
원문(sentence, 수정금지):  홍길동님 삼성전자 강남지점에서 일해요
라벨(annotation, 편집):    {{NAME|홍길동}}님 {{ORG|삼성전자}} {{ADDRESS|강남}}지점에서 일해요
```

규칙:
- 형식: `{{TYPE|텍스트}}`  (TYPE = NAME / ADDRESS / ORG)
- **조사·호칭·직책은 마크업 밖**: `{{NAME|홍길동}}님`, `{{NAME|김}}과장` (직책 분리)
- **본문 글자 수정 금지** — 마크업 기호만 추가. (reconcile 이 본문훼손 검출)
- 겹치지 않게. 가장 긴 의미단위로 (주소는 full span).
- PII 없으면 그대로 둔다 (마크업 0개 = 정상).

---

## 3. 챗 도메인 주의 (in-domain 특화)

- **공인·역사 인물 제외**: 뉴스와 달리 챗엔 유명인 언급 많음. "손흥민 경기 봤어" → 손흥민 라벨 X (공인).
- **1·2인칭 비식별 제외**: "저는", "우리 회사" → 라벨 X.
- **상담/의료 챗**: "23세 여자가 내원" 류 가상 환자 묘사 → 실명 없으면 NAME X. 병원명 실명이면 ORG.
- **모호 시 recall 우선**(label_guide_v1 §4) 하되, 공인/일반명사는 명확히 제외.
- 합성 구조형 PII 문장(여권/카드 등)은 annotator 파일에 **안 들어감** (자동정답).

---

## 4. 2인 독립 + 조정 원칙

1. 민우·양유상이 **서로 안 보고** 각자 `annotator_<이름>.jsonl` 채운다.
2. `reconcile.py` 가 span-exact 일치도(F1) + 불일치 목록 산출.
3. **합의 span = 자동 gold**. 불일치(`disagreements.jsonl`)만 둘이 모여 조정.
4. 일치도 < 0.70 이면 가이드 합의 부족 신호 → 본 문서 보강 후 재라벨.
5. 조정 결과를 반영해 최종 `eval_indomain_gold.jsonl` 확정.

---

## 5. raw PII 원칙

- 후보는 합성 + 공개 weak 코퍼스만. 실제 사용자 PII 금지 (label_guide_v1 §6).
- 평가 결과 보고 시 raw 텍스트 ❌, span offset·type·F1만 ✅.
