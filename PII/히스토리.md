## 주요 결정 로그

| 날짜 | 결정 |
|------|------|
| 2026-04-21 | **4-way 비교 레이어 구성:** Layer 0(Regex), 1(Claude API), 2(LiteLLM), 4(Custom)로 비교 구성. Layer 3는 예약. 이유: 순수 정규식 베이스라인 vs API 기반 vs 프레임워크 기반 vs 커스텀 구현의 명확한 대조를 위해. |
| 2026-04-21 | **validity-first 퍼저 전략(v4):** 무작위 생성 대신 한국 실제 규칙(주민번호 체크섬, 계좌번호 구조)에 맞는 유효한 PII를 우선 생성. 이유: 유효하지 않은 형식은 실제 모델이 오탐하지 않아 FP 분석 의미 감소. |
| 2026-04-21 | **Phase 1 McNemar 검정 도입:** 두 감지기 간 성능 차이의 통계적 유의성 검증. 이유: 단순 F1 차이는 샘플 크기에 따라 우연일 수 있음. |
| 2026-04-23 | **framing 재정립 (보호엔진 → 보강계층):** 가드레일을 "독립 보호 시스템"이 아닌 "기존 모델의 보강 레이어"로 포지셔닝 변경. 이유: Azure Content Safety와의 비교 결과 단독 시스템 주장이 무리임을 확인. |
