## 9. 논문 반영 권고

- 기존 `정상 텍스트 50건 중 1건(2.0%)` 문장은 유지하되, 본 addendum 결과를 반영하려면 `추가 clean-pass 10k synthetic normal corpus에서는 0건(0.00%)`이라고 별도 문장으로 쓰는 것이 안전하다.
- `0.00%`를 실제 운영 오탐률로 일반화하지 말고, synthetic 정상 입력에서의 확장 안정성 확인으로 해석해야 한다.
- KR_semantic 주장은 category별 표를 근거로 강화할 수 있다. 특히 prescription, allergy, degree, company, blood, dept, diagnosis에서 Layer 0의 기여가 크다.
- 변이형 payload 주장은 family별 결과로 방어하고, Unicode/format 계열 일부는 D 구성의 보완 효과로 설명하는 편이 정직하다.
- full-stack latency 반복 재호출은 관리형 API 비용/상태 의존성이 있으므로, 본 논문에는 `저장된 동일 10k 조건 재집계`와 `local Layer 0 반복 측정`을 분리해 명시하는 것이 좋다.

## 10. 한계

- 10k PII payload와 clean-pass corpus 모두 synthetic 중심이다. 실제 고객상담, 민원, 의료, 금융 로그의 표현 분포와 문맥 길이를 대표하지는 않는다.
- 관리형 API 구성의 latency는 서비스 상태, region, gateway 설정에 영향을 받는다.
- clean-pass 0건은 강한 신호지만, 정상 입력 generator가 배제한 PII-like pattern 밖의 도메인별 애매한 문장에 대해서는 별도 검증이 필요하다.
