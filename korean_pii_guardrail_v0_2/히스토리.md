## 주요 결정 로그

> 형식: **상황** 에서 **문제** 에 직면하여 **A 대신 B를 선택**했다. 목표는 **결과**이며 **트레이드오프**를 감수한다.
> 참고: [Y-statement ADR 형식](https://adr.github.io/) 기반

| 날짜 | 결정 |
|------|------|
| 2026-05-15 | **span resolver 위치 결정:** M6에 독립 모듈로 분리. 각 감지기가 독립적으로 span을 생성하고 나중에 한번에 정렬하는 방식 채택. 대안(각 감지기가 직접 중복 제거)은 감지기 간 의존성을 만들어 테스트가 복잡해지므로 기각. |
| 2026-05-15 | **HMAC keyring 방식 채택:** 감사 로그 무결성을 위해 단순 해시 대신 HMAC-SHA256 keyring 사용. 이유: 단순 해시는 로그 위변조 감지 불가, HMAC은 키 없이는 서명 위조 불가. 트레이드오프: keyring 관리 복잡도 증가. |
| 2026-05-21 | **Gradio 데모 탭 구조 결정:** 단일 분석 / 파이프라인 해부 / 배치 분석 3탭으로 구성. 이유: 단일 탭은 각 사용 목적(빠른 검증 / 디버깅 / 대량 측정)을 하나의 UI에 담기 어려움. 트레이드오프: 초기 학습 곡선 증가. |
| 2026-05-22 | **데모에서 raw PII 서버 사이드 보관:** 배치 분석 시 raw 텍스트를 클라이언트에 전송하지 않고 서버에 보관. 이유: 브라우저 콘솔/네트워크 탭을 통한 PII 노출 위험 제거. |
| 2026-05-25 | **릴리즈 게이트 FP 강화:** 인명 복합명사 FP 분석 후 threshold를 0.85→0.90으로 상향. 이유: `삼성전자이재용` 패턴에서 `이재용`이 단독 감지되는 경우 증가. |
